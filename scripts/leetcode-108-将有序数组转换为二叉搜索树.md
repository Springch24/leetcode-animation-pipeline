# LeetCode 108 — 将有序数组转换为二叉搜索树

## 标题
面试官给你一个有序数组让你转成二叉搜索树，90%的人随便取第一个当根。面试官说——树不平衡，挂了。

## 开场钩子（0-8s）
面试官给你一个有序数组[-10,-3,0,5,9]，让你转成二叉搜索树。90%的人随手取第一个元素当根，结果建出来的树歪成一条链——面试官直接说"回去等通知吧"。

## 题目描述（8-18s）
LeetCode 108 将有序数组转换为二叉搜索树。给你一个升序数组，把它转成一颗高度平衡的二叉搜索树。高度平衡的意思是：任意节点左右子树高度差不超过1。

## 解法核心（18-50s）
关键就四个字——**二分取中**。

每次取数组中间的元素当根节点，左边一半递归建左子树，右边一半递归建右子树。

比如 [-10, -3, 0, 5, 9]：
- 中间是 0，0 当根
- 左边 [-10, -3] 取中间 -3 当左子
- 左边 [] → null，右边 [] → null
- -3 左子是 -10，-10 左右都空
- 右边 [5, 9] 取中间 5 当右子
- 9 是 5 的右子

这样建出来的树天然平衡，高度 O(log n)。

## 动画描述（50-65s）
- 场景1：数组 [-10, -3, 0, 5, 9] 在屏幕上排开
- 高亮中间元素 0，0 跳到上方变成根节点
- 数组分成两半，[-10,-3] 高亮中间 -3，跳上去连到 0 的左指针
- [5,9] 高亮中间 5，跳上去连到 0 的右指针
- -10 连到 -3 左边，9 连到 5 右边
- 最终展示完整平衡二叉搜索树

## 复杂度（65-72s）
- 时间 O(n)，每个节点跑一次
- 空间 O(log n)，递归栈深度即树高

**"有序数组建平衡BST，中间元素就是根。"**
