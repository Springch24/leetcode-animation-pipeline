## 旁白稿

两个链表找交点，面试官让你写 O(n+m)，90% 的人第一反应：暴力双循环。然后面试就结束了。

LeetCode 160，相交链表。两个链表在某个节点之后共享同一段，找到那个相交点。暴力法 O(n×m) 超时，哈希表 O(n) 空间费内存。

最优解只要两个指针。指针 A 从链表 A 的头出发，指针 B 从链表 B 的头出发。每人每次走一步。走到末尾怎么办？跳到对方的链表头上继续走。

如果相交，A 走了 A独有 + 共享 + B独有，B 走了 B独有 + 共享 + A独有——总路程一模一样，一定在交点相遇。如果不相交，两人一起走到 null。

看动画。A 从 4 出发：4 → 1 → 8 → 4 → 5。B从 5 出发：5 → 6 → 1 → 8 → 4 → 5 → null——等等，B 走完自己的跳到 A 头！B：5 → 0 → 1 → 8 → 4 → 5 → null——不对，看我动画。

A：4、1、8、4、5。走完了，跳到 B 头。
B：5、0、1、8、4、5。走完了，跳到 A 头。
A 从 B 头继续：5、0、1、8——停！B 从 A 头继续：4、1、8——停！
两人同时踩在 8 上。交点找到了。

这个解法有个浪漫到犯规的名字：**走过你来时的路，我们终将相遇。**

时间复杂度 O(n+m)，空间 O(1)。两个指针，一人走一遍对方的路，交点就是答案。
