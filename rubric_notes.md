## 当前评分维度 (0-5) — v2 已校准（8维度）

| 维度 | 缩写 | 权重 | 含义 | 典型信号 |
|---|---|---|---|---|
| Animation Clarity | AC | 2.0 | 动画演示清晰度 | 评论"看懂了"；高完播率；每个算法状态变化有视觉反馈 |
| Hook Power | HP | 2.0 | 开头钩子力度 | 前5秒制造悬念/痛点/反差；标题触发焦虑或好奇心 |
| Teaching Rhythm | TR | 1.5 | 教学节奏（含时长最优区间） | 动画速度 vs 旁白同步度；节奏紧凑无拖沓；75-85s最优 |
| Key Insight | KI | 1.5 | 关键洞察（一句话总结能力） | 有无「看完就记住」的金句；是否解释了核心trick/直觉 |
| Audience Breadth | AB | 1.2 | 受众广度（题目面试频率） | 题目在面试中的出现频率；是否跨语言/岗位通用 |
| Actionability | ACT | 1.2 | 可操作性（看完能否自己写） | 步骤分解清晰度；收藏率；观众是否能看完就动手 |
| Code Readability | CR | 1.0 | 代码可读性 | 字体大小、关键词高亮、代码布局美观度 |
| Visual Style | VS | 1.0 | 画面风格统一感 | 色调、排版、动画风格统一且有辨识度 |

**v2 综合分公式**：

```
composite = (AC×2.0 + HP×2.0 + TR×1.5 + KI×1.5 + AB×1.2 + ACT×1.2 + CR×1.0 + VS×1.0) / 11.4 × 2.0
```

**满分 10 分**（除以权重总和 11.4，乘以 2.0）。

**v2 与 v1 核心差异**：
- HS→HP：钩子从 1.5→2.0（12 样本充分验证，抖音滚动流钩子决定留存）
- RM→TR：教学节奏扩展，加入时长最优区间信号（75-85s 赞播比最高）
- **新增 KI**（关键洞察 权重 1.5）：一句话总结能力驱动赞播比
- **新增 ACT**（可操作性 权重 1.2）：看完能不能写，收藏率 proxy
- ED 移除：被拆分为 KI + ACT，更具体可衡量
- 总权重和从 9.4→11.4

## 各维度详解

### AC — Animation Clarity（动画清晰度，权重 ×2.0）
*算法步骤是否被动画正确、流畅地演示出来？*

- **0** — 纯静态画面，只靠旁白讲
- **3** — 有动画但节奏不对（说完了画还没动，或者画比话快太多）
- **5** — 动画和算法步骤同步推进，每个状态变化都有视觉反馈

### HP — Hook Power（钩子力度，权重 ×2.0）
*前 5 秒是否让人想继续看？标题是否触发焦虑/好奇心？*

- **0** — 通用开场（"大家好今天讲一道题"），标题平淡
- **3** — 标题有设计感，或开头抛出一个具体问题
- **5** — 前 5 秒制造悬念或痛点（如"90% 的人第一次都写错""面试官会直接挂你"），让人非看不可

### TR — Teaching Rhythm（教学节奏，权重 ×1.5）
*动画推进速度和旁白讲解速度是否匹配？整体节奏是否紧凑？目标时长 75-85 秒。*

- **0** — 严重脱节（旁白说完画面还是静态；或画面切换太快跟不上）；或时长明显超标（>120s）
- **3** — 大致同步，偶有偏差
- **5** — 音画完美同步，每个动画精准配合每句话，节奏紧凑无拖沓

### KI — Key Insight（关键洞察，权重 ×1.5）
*是否有一个让观众「看完就记住」的金句或核心 trick？是否解释了「为什么」而不仅是「怎么做」？*

- **0** — 纯念代码，没有任何洞察
- **3** — 有解释逻辑但不精炼，无记忆点
- **5** — 有一句话就能总结的核心技巧（如「有环必相遇」「保存→反转→前进」「代码只有五行」），观众愿意截图保存

### AB — Audience Breadth（受众广度，权重 ×1.2）
*这道题在面试/搜索中有多高频？受众面有多广？*

- **0** — 冷门题，几乎没人搜
- **3** — LeetCode Hot 100 中段题目
- **5** — 面试 Top 10 高频题（反转链表、两数之和等），或入门必刷题（移动零），跨语言/跨岗位通用

### ACT — Actionability（可操作性，权重 ×1.2）
*看完能不能自己写出来？步骤是否清晰可复现？*

- **0** — 看完完全不知道怎么写
- **3** — 大致懂了思路但细节不够
- **5** — 步骤分解清晰，每个状态变化都有标记，观众看完能自己动手实现

### CR — Code Readability（代码可读性，权重 ×1.0）
*画面上的代码是否清晰、字体大小合适、关键词高亮正确？*

- **0** — 代码模糊、字体太小、没有高亮
- **3** — 代码清晰，基础高亮
- **5** — 代码布局美观，关键词高亮层次分明，观众愿意截图保存

### VS — Visual Style（画面风格，权重 ×1.0）
*整体画面的色调、排版、动画风格是否统一且有辨识度？*

- **0** — 杂乱、没有统一设计
- **3** — 有统一配色和字体
- **5** — 视觉系统成熟、一眼能认出品牌（深色主题 + 统一配色）

## 重大跨视频观察（≥2 样本支持）

1. **链表 + 简单题 = 爆款**：4 条链表×简单题视频平均 3,266 播放（反转3,696 + 环形675 + 两数相加3,151 + 相交5,241），碾压其他标签组合
2. **焦虑钩子公式持续有效**："90% 的人……" 标题在 4+ 条视频中均带来高于平均的播放
3. **时长与赞播比负相关**：75-85s 最优区间，超过 120s 赞播比显著下降
4. **互动引导完全失效**：当前 CTA 形式（"评论区见"）零效果，需策略变革

---

## 规律沉淀区（高置信度，打分前必看）

- **ACT 维度对于简单题应偏高**：简单题天然可操作性强，ACT≥3 是基础预期
- **HP 维度对于非 Top10 题目更难拿高分**：题目本身不够知名时，钩子需要更用力
- **时长 >100s 自动触发预警**：不扣 composite 分，但在 predict 输出中标注预警等级
- **赞播比是内容密度的独立指标**：composite 高 ≠ 赞播比高（lc206: 8.77 composite, 1.52% vs lc141: 8.11 composite, 2.81%）
- **AB 维度不只看题号**：移动零虽非 Hot 100 Top10，但作为入门必刷题 AB 等效 Top5
- **收藏量由 ACT×KI 联合驱动**：同时具备高可操作性和高金句价值的视频收藏爆发

## v2.1 → v2.2 bump 记录 (2026-06-01)

### Bucket 重校

基于 N=14 重算 percentile：
- p30=574, p60=3777, p85=5387, p95=6135
- 当前边界：583/3932/5397/6265
- **决策**：不改。变化 <5%，N=14 太小不值得动边界。等 N≥20 再重校。

### Spearman ρ

- v2.1 (N=10): 0.891
- v2.1 (N=14): **0.8022** — 下降 0.089
- 主因：lc234 (回文链表) rank 偏差 +6（composite 8.25 → 播放 489，被封面忘传拖累）
- 次因：lc128/lc283 rank 偏差 -4（中等难度播放不稳定）
- ρ=0.8 在 14 样本下仍为强相关（p=0.0006），继续使用 v2.1 公式

### 赞播比 discovery

- **KI 不能独立预测赞播比**：KI=5 简单题 2.81% (lc141) vs 0.75% (lc160)，差异 3.7x
- **难度是赞播比的隐藏调节变量**：
  - 简单题赞播比 1.0-2.8%
  - 中等题赞播比 0.8-1.5%
- 建议 v2.2 加难度对赞播比的修正系数（不做公式更改，只在 predict 输出里加难度标注）

### 2s跳出率信号强化

- lc020 39.62% + lc019 39.87%，连续两条 <40%
- <40% 的 2s 跳出率是爆款前兆（两条都≥3,932 播放）
- 此后每做视频都要关注前 2 秒必须有动画（不能黑屏+文字）

### 待 v2.2 决策

- [ ] 难度档位对赞播比的调节系数
- [ ] lc234 封面忘传是否应该标记异常值（从 calibration pool 移除？）

### 核心发现：标签是播放的第一决定因素

| 标签 | 条数 | 均播 | 范围 |
|---|---|---|---|
| 链表（含设计关联） | 6 | 3,262 | 1,454-5,170 |
| 堆/单调栈 | 2 | 2,140 | 1,519-2,760 |
| 栈（纯） | 2 | 199 | 131-266 |
| 贪心/股票 | 3 | 460 | 454-472 |

**结论**：抖音算法视频的受众根本不在乎「单调栈」「栈」「贪心」这些标签。
- **链表=安全牌**：即使用中等题，链表标签仍能均播3,000+
- **栈=死亡标签**：lc084/lc394两条合计不到400播放
- **贪心=冷门**：三道股票题（lc121v1/v2 + 可能与之前lc121重复）全部<500

### Hook 必须用「面试官说」反问结构

- lc121v2：「给你一个价格数组，算最大利润」→ 2s跳出55.71%，立即死亡
- lc019：「面试官让你删倒数第N个，你说扫两遍。面试官说——一遍过」→ 2s跳出39.87%，播放5,435
- **差距 = 陈述句 vs 反问句**

### Hook 2s内必须制造认知冲突

对比同是栈标签：
- lc394 (266播放): Hook「90%说数括号，嵌套一来就懵了」→ 有冲突但受众不关心
- lc155 (4,150播放): Hook「getMin也要O(1)」→ 设计递进，让人好奇怎么做到
- **冲突必须关乎受众已有的认知**，不是凭空制造

### 困难题在 cold-start 阶段禁止

- lc084 (131) 是本阶段唯一困难题，直接触底
- 即使lc023 (1,519) 也不如中等链表题
- **N<50样本前，禁止做困难题**

- lc121v2: 2s跳出55.71%、完播0.65% → 播放472
