深度学习的十年，是一场没有硝烟的架构战争。

2012 年，AlexNet 在 ImageNet 上一战成名，CNN 成为计算机视觉的绝对王者。2015 年，LSTM 统治了机器翻译和语音识别，RNN 是序列建模的唯一答案。然后 2017 年，一篇叫《Attention Is All You Need》的论文出现了。

Transformer 没有卷积，没有循环，只有注意力。它用一场范式革命，终结了 CNN 和 RNN 长达五年的双雄时代。

回头看，这三次架构变迁不是简单的「算法升级」，而是**三次对信息本质的重新理解**。

---

## 一、CNN：空间局部性优先

CNN 的核心思想朴素而强大：图像里相邻的像素有关系，远处的没关系。一个猫的耳朵和它的眼睛靠在一起，和图片左下角的背景没关系。

于是有了卷积核——一个滑动的小窗口，只在局部区域计算，所有位置共享同一套参数。

```
输入图像 224×224×3 → 150,528 个像素
全连接第一层 1000 个神经元 → 1.5 亿个参数 → 直接训炸
CNN 3×3 卷积核 64 个 → 1,728 个参数 → 轻松训练
```

CNN 还天然适合堆叠。两个 3×3 卷积核的感受野等于一个 5×5 的，但参数更少，非线性更多。VGG 把这一套玩到极致，ResNet 加上残差连接堆到 152 层。图像识别的准确率从 70% 一路飙升到 95%+。

但 CNN 有一个根本性的局限：**它天生不擅长序列。** 文本、语音、时间序列——这些数据的结构不是「位置相邻」而是「顺序依赖」。CNN 可以强行用一维卷积处理文本，但它不知道「我」和「钥匙」之间隔着「今天早上出门的时候发现」意味着什么。

---

## 二、RNN/LSTM：时间顺序优先

RNN 的设计哲学正好相反：序列里的每个词不仅要看自己，还要看之前发生了什么。

```
输入: 「我 今天 早上 出门 的 时候 发现 钥匙 忘记 带了」
h₁ = tanh(W·「我」)
h₂ = tanh(W·「今天」 + h₁)
h₃ = tanh(W·「早上」 + h₂)
...
h₁₀ = tanh(W·「带了」 + h₉)  ← h₁₀ 包含了整句话的信息
```

这个设计很优雅。但它有一个致命的缺陷：**梯度消失。**

当序列很长时，反向传播要乘几十次同样的权重矩阵。如果权重小于 1，梯度指数衰减到几乎为零——模型学不到「我」和「带了」之间的关系。这就是为什么早期 RNN 只能处理短句子，长句子直接懵。

LSTM 用一个巧妙的设计解决了这个问题：**细胞状态（Cell State）。** 它在隐藏状态之外加了一条「高速公路」，信息通过加法传递而不是乘法。梯度不再指数衰减，模型终于能记住 100 个词之前的「我」。

LSTM 统治了 2015-2017 年的 NLP 世界。机器翻译、语音识别、文本生成——全是 LSTM 的天下。直到 Transformer 出现。

---

## 三、Transformer：全局依赖优先

Transformer 的哲学和前面两个完全不同：**不要一层层传递信息，每个词直接看所有词。**

Self-Attention 的核心操作是一次矩阵乘法。拿一句话的所有词向量，和它们自己的转置相乘，得到一个「关注度矩阵」——每个词对每个词的注意力分数。然后用这个分数加权所有词的语义信息。

```
「The cat sat on the mat」
对于「sat」：
  The     cat     sat     on      the     mat
  0.05    0.30    0.10    0.25    0.05    0.25
```

`sit` 同时看了句子里的每一个词，自己决定该关注谁。没有衰减，没有遗忘，没有「距离」概念。

这个简单到近乎粗暴的设计，解决了 CNN 和 RNN 各自最大的问题：CNN 不擅长序列，RNN 记不住远处。Transformer 两者通吃——既能并行计算（像 CNN），又能捕捉长距离依赖（比 RNN 强得多）。

代价是计算量。Self-Attention 的复杂度是 O(n²)——句子长度翻倍，计算量翻四倍。所以 Transformer 在短文本上不如 RNN 高效，但在 GPU 并行加持下，这个代价完全可以接受。

---

## 四、为什么是 Transformer 赢了

这场架构战争打了十年，最后 Transformer 胜出，不是因为它更「聪明」，而是因为一个更根本的原因：**GPU 喜欢矩阵乘法。**

CNN 的卷积可以展开成矩阵乘法。RNN 的串行循环不行——每一步都必须等上一步算完。这就是 RNN 的死穴：无论你堆多少块 GPU，训练速度都被串行步骤锁死。

Transformer 的 Self-Attention 就是一次矩阵乘法。GPU 最擅长的事。所以 Transformer 可以轻松扩展到百亿参数，而 RNN 在千万参数级别就开始吃力。

**硬件决定了算法的命运。** 这不是什么深刻的哲学洞察，就是赤裸裸的工程现实。

---

## 五、架构战争教会我们的事

回头看这十年，有一条规律反复出现：**每次范式转移，都是因为新架构解决了上一代架构的「不擅长」。**

CNN 不擅长序列 → RNN 来了。RNN 不擅长长序列 → LSTM 来了。LSTM 不擅长并行 → Transformer 来了。

Transformer 统治了 2017 年至今。但它的 O(n²) 复杂度在超长序列面前已经开始吃力。Mamba、RWKV 这些新架构正在尝试用状态空间模型替代注意力——也许下一次范式转移已经在酝酿。

**架构战争从来没有真正的「赢家」。只有在下一次颠覆到来之前，暂时领先的那一个。**
