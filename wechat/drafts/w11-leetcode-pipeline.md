过去两个月，我用 AI 做了 35 条 LeetCode 算法动画视频，发在抖音和 B站。

每条视频 75-85 秒，竖屏 1080×1920，旁白 + HTML 动画渲染。内容形态很垂直：面试算法题的可视化解法。

这篇文章是完整生产流程的拆解。每一步的模板、公式、经验全部公开，你也可以复用。

---

## 一、为什么是 LeetCode 算法动画

选这个方向有三个理由：

**第一，面试焦虑是永恒的需求。** 「面试官让你写反转链表，90% 的人会卡在这道题」——这是真实的用户痛点，不是伪需求。

**第二，算法需要可视化。** 文字解释递归、双指针、滑动窗口是反直觉的。动画让观众「看到」算法的运行过程，理解深度远超文字。

**第三，内容可规模化。** LeetCode 有 3000+ 道题，选题几乎是无穷无尽的。标准化生产流程后，可以批量生产。

---

## 二、选题策略

35 条视频跑下来，播放量最高的组合是**链表 × 简单题**——平均 3266 播放。中等难度的链表题、简单难度的二叉树题也还行，但已经明显不如前者。

选题优先级排名：

1. 链表 × 简单（黄金组合）
2. 链表 × 中等
3. 二叉树 × 简单
4. 数组/双指针 × 中等

代码相关的题（字符串处理、正则匹配）不适合做动画，纯静态代码展示对观众没有吸引力。

## 三、脚本模板（75-82 秒）

每条视频的脚本结构是固定的：

- **0-3 秒 钩子**：「面试官让你写 XX 题，90% 的人……」——焦虑钩子，必须第一帧出现
- **3-11 秒 标题**：LeetCode 题号 + 中文题名
- **11-30 秒 题目可视化**：用动画展示题目描述，让观众「看到」输入→输出
- **30-65 秒 解法动画**：核心部分。算法每一步的状态变化都要有视觉反馈
- **65-72 秒 复杂度**：时空复杂度 + 一句话解释
- **72-78 秒 金句收尾**：一句话能记住的核心技巧。比如环形链表的金句「有环必相遇」
- **78-82 秒 CTA**：引导点赞收藏

金句是脚本的灵魂。它的标准：**一个人看完视频后，如果只记住一句话，就是这句。**

## 四、动画制作

桌面端用 GSAP 写 HTML 动画，然后通过 HyperFrames 渲染成 MP4。移动端竖屏 1080×1920，字体用系统字体 fallback（PingFang SC → sans-serif），不依赖外部 CDN。

关键细节：

- **钩子前置**：第 0 帧必须有完整钩子文字，不依赖飞入动画。降低 2 秒跳出率
- **每个状态变化有视觉反馈**：指针移动、节点高亮、变量更新——这些是「可看见的思考」
- **6 个场景切换**：钩子→标题→题目→解法→复杂度→CTA。转场用 GSAP timeline
- **注册全局 timeline**：`window.__timelines`，HyperFrames 需要它来控制渲染

## 五、TTS + 渲染

旁白文本写到 `narration.txt`，用 edge-tts 生成 MP3：

```bash
python3 -m edge_tts --voice zh-CN-YunyangNeural \
  --file narration.txt --write-media narration.mp3
```

视频渲染用 HyperFrames：

```bash
npx hyperframes render . -o "lc141-环形链表.mp4" -f 30 -q high
```

音频合并用 ffmpeg：

```bash
ffmpeg -i video.mp4 -i narration.mp3 \
  -c:v copy -c:a aac -map 0:v:0 -map 1:a:0 -shortest -y final.mp4
```

## 六、发布 + 评论区运营

抖音简介三段式：钩子 + 金句 + 标签。评论区置顶发代码卡片（含完整 Python 代码），配引导话术如「你第一次做用了双指针还是哈希表？」

发布后 T+3 天复盘，用播放量、平均时长、赞播比、收藏率四个指标评估。链表 × 简单题的收藏率最高，说明「可操作性」和「关键洞察」联合驱动了收藏行为。

## 七、这 35 条视频教我的三件事

**钩子的力量远超你的想象。** 第一条视频用「大家好今天讲一道题」，播放量 440。换成「面试官让你写反转链表，90% 的人会卡在这」后，同类型内容播放量飙到 3000+。开头 2 秒决定 80% 的留存。

**标准化流程才能规模化。** 从选题到脚本到动画到发布，每一步都有固定模板。模板不是为了限制创意，是为了把机械操作交给 AI，把创意留给选题和金句。

**75-85 秒是甜点时长。** 更短讲不清楚，更长观众没耐心。85 秒附近开始有轻微跳出，超过 100 秒跳出明显加速。

## 八、你可以怎么开始

如果你也想做类似的垂类内容，建议从这三步开始：

1. **选 5 道你最熟悉的题**，先把流程跑通，不看数据
2. **找到你的黄金题型**，根据播放量锁定方向
3. **建立模板和 AI 工作流**，把生产时间压缩到 1 小时内
