## 三、降本策略（不用花自己的钱）

### 策略 1：用免费额度活过冷启动期

| 服务 | 免费额度 | 够用到 |
|------|----------|--------|
| Gemini Flash | 1500 次/天免费 | 永远够（Flash 够用） |
| Gemini Pro | 50 次/天免费 | 日活 500 以内 |
| 微信云开发 | 5GB 存储 + 5万次调用/月 | 日活 500 以内 |
| Cloudflare Workers | 10 万次/天免费 | 永远够 |

**结论：日活 500 以内，AI 成本 ¥0。**

### 策略 2：用户自己带 Key

```
┌──────────────────────────────────────┐
│  「粘贴你的 Gemini API Key（免费获取）」  │
│  → 你的 Key 只存在你本地，我们不上传     │
│  → 或者直接用我们提供的免费额度（有限）   │
│  → 想无限用？自己申请 Key，30 秒搞定     │
└──────────────────────────────────────┘
```

用户自己申请 Gemini API Key（免费），填进小程序。高级功能用他们的 Key，基础功能用你的免费额度。**你把 AI 成本完全转移给用户或 Google。**

### 策略 3：限制免费次数，解锁靠行为

| 功能 | 免费额度 | 如何解锁更多 |
|------|----------|-------------|
| AI 代码批改 | 3 次/天 | 连续打卡 7 天 → 10 次/天 |
| AI 模拟面试 | 1 次/天 | 邀请 3 个好友 → 5 次/天 |
| 付费题库 | 每周 1 题 | 看激励广告 → 解锁 |

用户行为 = 免费额度，不想行为 = 自己带 Key 或看广告。

### 策略 4：收入覆盖成本后才扩展

```
成本线 ← 收入超过这条线才花钱扩展
  │
  │  ¥0 ────── 冷启动（纯免费额度）──────────→ 日活 0-500
  │  ¥50/月 ── 开始有广告收入 ──────────────→ 日活 500-2000  
  │  ¥200/月 ─ 收入覆盖成本，加 Pro 模型 ───→ 日活 2000+
```

## 七、资质 & 合规（小程序上线需要什么）

### 7.1 微信小程序

| 类型 | 需要什么 | 费用 | 审核时间 |
|------|----------|------|----------|
| **个人主体**（推荐起步） | 身份证 + 微信号 | ¥0 | 1-3 天 |

**个人主体就够用！** 你的小程序一开始就是刷题打卡 + AI 批改，不属于「商业服务」类目，不需要营业执照。

个人主体 vs 个体工商户：

```
个人主体：
  ✅ 免费，身份证就能注册
  ✅ 够用到日活 5000+
  ❌ 不能开通微信支付（但你可以用「激励广告」变现）
  ❌ 不能使用部分类目（直播、电商等）

个体工商户：
  ✅ 可以开通微信支付
  ❌ 要办执照（线上 5 分钟，但每年要年报）
  ❌ 可选：刻章备案 ¥200-500（不是必须的）
```

**建议：先用个人主体上线，等需要支付功能了再升级个体户。**

## 九、合规注意事项

### 你需要做的：
- 用户协议 & 隐私政策（套模板就行，5 分钟）
- 不收集用户敏感信息（手机号可选，不强制）
- AI 生成内容标注「由 AI 生成，仅供参考」

### 你不需要做的：
- 没有 App Store 审核（小程序审核比 App 松多了）
- 没有软著（著作权在你代码里就已经自动有了）
- 没有 ICP 备案（微信小程序走平台备案，不需要自己跑管局）
