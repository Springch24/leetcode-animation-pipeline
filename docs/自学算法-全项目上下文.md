## 四、视频生产流水线

### 4.1 技术栈
- **动画**：GSAP 3.14.2（CDN）
- **渲染**：HyperFrames CLI v0.6.x → MP4
- **配音**：edge-tts（微软免费 TTS），男声 `zh-CN-YunyangNeural`
- **封面**：HyperFrames 渲染 2 帧 MP4 → ffmpeg 抽 PNG
- **字体**：Noto Sans SC + JetBrains Mono（Google Fonts，HyperFrames 自动缓存）

### 4.2 视频结构（75-85 秒）
| 时间段 | 内容 |
|--------|------|
| 0-5s | 标题动画 |
| 5-22s | 题目描述 + 示例可视化 |
| 22-60s | 解法动画（代码逐行 + 数据模拟） |
| 60-75s | 复杂度分析卡片 |
| 75-82s | CTA 结尾 |

### 4.3 生产一条视频的步骤
```
1. 准备题目数据（编号、标题、难度、描述、标签、代码）
2. 我（AI）生成 index.html（GSAP 动画时间线）
3. 我（AI）生成 narration.txt（中文旁白）
4. python3 -m edge_tts → narration.mp3
5. ./cover-template/generate-cover.sh → 封面 PNG
6. npx hyperframes render → 视频 MP4（无音频）
7. ffmpeg 合并视频+音频 → 最终 MP4
```
注意：HyperFrames 渲染时需要 `export PATH="$HOME/.local/bin:$PATH"`（ffmpeg 在 `~/.local/bin/`）

### 4.4 关键约定
- 视频 HTML 的 `<body>` 必须带：`data-composition-id="test" data-width="1920" data-height="1080"`
- 脚本最后必须注册：`window.__timelines = { test: tl };`
- 不需要 `<audio>` 元素，音频后期用 ffmpeg 合并（避免 lint 警告）
- 封面 HTML 必须带：`data-composition-id="cover" data-width="1920" data-height="1080" data-duration="1"`
- 封面渲染用 `-f 2`（2fps），`data-duration="1"`（否则 staticDuration=0 会失败）
- 深色主题：`#0D1321` 背景，`#FCA311` 强调色，`#F0EBD8` 主文字

### 4.5 代码风格
- 代码用 JetBrains Mono 西文
- 其他界面文字用 Noto Sans SC 中文
- 代码高亮：Python 关键字 `#569CD6`，函数 `#DCDCAA`，数字 `#B5CEA8`
